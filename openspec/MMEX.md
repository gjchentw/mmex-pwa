# MMEX C++ Desktop Source (`mmex/moneymanagerex/`)

## Overview

- **Language**: C++17
- **GUI Framework**: wxWidgets 3.x
- **Database**: SQLite3 via wxSQLite3
- **Build System**: CMake
- **JSON**: RapidJSON
- **Scripting**: Lua (embedded via LuaGlue)
- **Charts**: ApexCharts.js (HTML/JS embedded in reports)
- **Networking**: libcurl
- **Other**: fmt (formatting), cgitemplate (HTML templates)

Each table defines:
- `Data` struct with column fields
- `Data_Set` (`std::vector<Data>`) with `to_json()`
- `Cache` and `Index_By_Id` for in-memory caching
- CRUD methods: `get()`, `save()`, `remove()`, `exists()`, `ensure()`
- Column comparison helpers: `DB_Column<V>` with operators (EQUAL, GREATER, LESS, etc.)

**Key file**: `DB_Table.h` — base class with 1286 lines of CRUD infrastructure, column definitions, JSON serialization, and SQL query building.

**Database upgrades**: `DB_Upgrade.h` defines incremental SQL upgrades from version 0 to 21, auto-generated by `sqliteupgrade2cpp.py`. Current latest version: 21.

### Layer 2: Business Models (`src/model/`)

24 model classes extending the `Model<DB_TABLE>` template. Each model wraps a DB_Table and adds business logic.

**Key file**: `Model.h` — base template providing:
- Singleton access via `instance(wxSQLite3Database*)`
- CRUD delegation to DB_Table
- `get_all()`, `find()`, `top_level()` query methods

**Key models**:

| Model | Wraps DB_Table | Business Logic |
|---|---|---|
| `Model_Account` | `ACCOUNTLIST_V1` | Account types (Cash, Checking, Credit, Loan, etc.) |
| `Model_Checking` | `CHECKINGACCOUNT_V1` | Transaction types (Withdrawal, Deposit, Transfer), statuses |
| `Model_Category` | `CATEGORY_V1` | Hierarchical categories |
| `Model_Payee` | `PAYEE_V1` | Payee management |

| `Model_Stock` | `STOCK_V1` | Stock/Share holdings |

**Pattern**: Models use `static instance()` singleton. Each `Full_Data` struct extends `Data` and adds joined fields (e.g., `Model_Account::Full_Data` includes the currency name).

## Data Flow

```
User Action
    ↓
Dialog/Panel (View)
    ↓
Model::instance(db) → query/CRUD (Business Logic)
    ↓
DB_Table::get/save/remove (SQL Generation)
    ↓
wxSQLite3Database → SQLite3 (Storage)
```

For reports:

```
User requests report
    ↓
Report class gathers data via Models
    ↓
htmlbuilder generates HTML + ApexCharts JS
    ↓
wxHtmlWindow renders in panel
```

## Application Lifecycle

1. **`mmGUIApp::OnInit()`** (`mmex.cpp`):
   - Parse command line
   - Initialize locale/i18n
   - Splash screen
   - Database selection (new/existing)
   - Call `mmGUIFrame` constructor
2. **`mmGUIFrame` constructor** (`mmframe.cpp`):
   - Set up wxAUI manager
   - Create panels (home page, accounts, reports, etc.)
   - Load options from DB
   - Set up file history
   - Register database update hooks
3. **Runtime**: Event-driven (wxWidgets events, menu, toolbar, panel interactions)
4. **Shutdown**: `mmGUIApp::OnExit()` — save settings, close DB

## Config / Options System

- `Option` class (singleton, `option.h/cpp`): caches all application settings from DB tables (`INFOTABLE_V1` + `SETTING_V1`)
- Dialog: `optiondialog` with tabs: General, View, Transactions, Attachments, Network, Home, Misc
- Per-machine settings stored in `mmexini.db3` (SQLite)
